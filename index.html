Background

HTML (HyperText Markup Language) is a scripting language. Though it is not actual programming, it is a type of SGML (Standard Generalized Markup Language). SGML variations are usually easy to learn and require little computer expertise.

History

HTML was developed by Tim Berners-Lee and popularized in 1992 with the introduction of the first graphical web browser, Mosaic. Both Netscape Navigator and Microsoft Internet Explorer are based on NCSA Mosaic's code. Now, the World Wide Web Consortium sets the standards for HTML. Browser companies such as Microsoft or Netscape generally follow these guidelines voluntarily.

Vocabulary

All HTML "commands" are contained within < (less-than) and > (greater-than) signs. Certain commands which change text or denote divisions in a document require closing "commands." They begin with a less-than sign and a forward slash (/) and end with a greater-than sign. The word immediately following the less-than sign and/or the less-than sign and forward slash is known as a tag. When referred to collectively, opening and closing tags are known as elements. All entities following the space after the tag are attributes.

Put more simply:
<TAG ATTRIBUTES>Something here</TAG>
The BLOCKQUOTE element

Style

It is a good idea to capitalize all text between less-than and greater-than signs. This helps readability and allows easy distinguishing between markup and content.

Nesting

Most elements can be nested within each other. Use common sense when nesting. You would not put a table inside a link. Elements must be closed in the order in which they were opened.

<B><I>Some text</I></B>, not <B><I>Some text</B></I>

Combining Attributes

When wanting to specify more than one attribute for one specific element, the attributes may be combined. This is best demonstrated with an example:

<FONT COLOR="RED"> + <FONT SIZE=4> = <FONT COLOR="RED" SIZE=4>

Document Outline

Basic Structure

The entire web page is contained within the HTML element. There are only two elements which are found directly beneath HTML, HEAD, the document header and BODY, everything else. Within HEAD, usually only the document TITLE is found.

More simply:
<HTML>
 <HEAD>
  <TITLE>Document Title</TITLE>
 </HEAD>
 <BODY>
  Everything goes here
 </BODY>
</HTML>

BODY Attributes

Some general properties which apply to the entire document are set within the <BODY> tag. These include default text color, default link color, background color, and background image.

Text Color

This sets the default text color for the document. If this attribute is not set, black is used.

<BODY TEXT="WHITE">

Link Color

This sets the default color of unvisited links in the document. If this attribute is not set, blue is used. (This is best left alone since most people are accustomed to blue links. Only change this if your color scheme requires it.)

<BODY LINK="RED">

Background Color

This sets the color shown behind all text and images in the document. It is best to always set this since different browsers have different default background colors (usually gray or white).

<BODY BGCOLOR="WHITE">

Background Image

This sets the image shown behind all text and images in the document. It will tile both vertically and horizontally. Even if a background image is used, it is a good idea to set a background color similar to the color of the background image. That way, while the image is loading, the background color will be shown.

<BODY BACKGROUND="clouds.jpg">

Important Information

Spaces

One strange thing you will notice about HTML is the way spaces are interpreted. No mater how many spaces you put in a document, only one is shown. For example, the code:

Something           here
will display in the browser as:
Something here

Use &nbsp; (nonbreaking space) to add multiple spaces. The following will produce the desired result:

Something &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; here

Breaks

Another similar strangeness of HTML is the way line breaks are interpreted. They are not at all. Each break is replaced with a single space:

a
b
c

displays as:

a b c

Use <BR> to force a line break. Using a<BR>b<BR>c will produce the desired result.

Groupings

Paragraphs

The P element is generally used to contain paragraphs. Paragraphs, like most elements, can be aligned either left, right, or center. By default, they are aligned left. As a good rule of thumb, never center paragraphs. The constant shifting of the left edge makes reading centered paragraphs difficult. Breaks are automatically placed before and after paragraphs.

<P>This paragraph will be aligned with the left side of the browser.</P>
<P ALIGN="RIGHT">This paragraph will be aligned with the right side of the browser.</P>

Divisions

The DIV element is a generic container that can hold anything. Usually a document is divided up into many divisions. When aligning a division (left, right, center), all elements within the division will be aligned. By default, divisions are aligned left. Breaks are automatically placed before and after divisions. Usage is identical to P.

Horizontal Rules

Horizontal rules (horizontal lines), HR, are generally used to separate the different parts of a page. By default, horizontal rules are centered and take up almost the entire screen. A break is automatically placed before and after the horizontal rule.

<HR>

The thickness of an HR can also be controlled. Set the attribute SIZE to the desired thickness in pixels.

<HR SIZE=5>

FONT Element

You will most likely use the FONT element more than any other element. Remember, all the attributes below can be set in one FONT to save space. The FONT element requires at least one of the three below attributes.

Size Attribute

Font sizes range from one to seven. One is the smallest. Seven is the largest. If a font size is not specified, three is used.

<FONT SIZE=1>Small text</FONT>

Small text

<FONT SIZE=7>Large text</FONT>

Large text

Color Attribute

Font colors can be any of the basic color names. The default font color is the color set in the BODY TEXT attribute.

<FONT COLOR="PURPLE">Purple text</FONT>

Purple text

Face Attribute

The FACE attribute determines what font will be used. By default, the font Times New Roman is used on almost all browsers. Some of the common fonts found on PCs and Macintoshes include Arial, Helvetica (Macintosh equivalent of Arial), Courier, and Verdana.

<FONT FACE="Helvetica">Helvetica font</FONT>

Helvetica font

Headings

Headings come in six sizes, one through six. The element used depends on the desired size. To create a level one heading (largest), use the H1 element. To create a level six heading (smallest), use H6. Breaks are automatically placed around headings. Headings should not be used to control the font size of text. See FONT for changing font size within a paragraph. Headings are used to denote new sections or subsections. In this section, the word "Headings" above is a level three heading. Use headings in decreasing order; do not skip numbers. Headings can be aligned left, right, or center. The default, as always, is left.

<H4 ALIGN="CENTER">Level four heading, centered</H4>

Level four heading, centered


Formatting

Bold

Strengthen the emphasis of a word or phrase with the B element. Bold, like other formatting elements, should be used sparingly.

<B>Bold text</B>

Bold text

Italic

In typewritten pages as well as web pages, italics (the I element) are a substitute for underline. Use italics to emphasize anything you would have underlined in writing.

<I>Italic text</I>

Italic text

Underline

Since underlining is replaced by italics, use the U element very rarely. Another reason to avoid underlining is the confusion it causes, since links also use underlined.

<U>Underlined text</U>

Underlined text

Teletype

The TT element is for making text fixed-width. It is supposed to look like a typewriter font and is usually used for citing code.

<TT>Teletype text</TT>

Teletype text

Superscript

The SUP element makes text smaller and shifts it upward. This is usually used in mathematics to show exponents.

x<SUP>2</SUP>

x2

Subscript

The SUB element makes text smaller and shifts it downward. This is primarily used in chemistry.

H<SUB>2</SUB>SO<SUB>4</SUB>

H2SO4

Links

All linking is done with the A element followed by the HREF attribute. The only variation throughout the below subsections is the value of the attribute HREF. Never use click here, here, or this as a link, ever. Not only are you making a mockery of the entire concept of hypertext, but also destroying any chances of the user being able to scan through your document.

Local

To link to another page on your web site, set the HREF to the name of the document.

<A HREF="homepage.html">my web site</A>

my web site

External

To link to an external site or page on an external site, use the same format as above, just replace the name of the document with the full URI (Uniform Resource Identifier) Always remember to include the http://.

<A HREF="http://www.lacey-chabert.com/">Lacey Chabert</A>

Lacey Chabert

E-Mail

The procedure is identical to make a link to e-mail someone. Replace the HREF value with mailto: plus the e-mail address. When clicked, the browser's e-mail subsystem will start and a message window will open with the To: address already specified.

<A HREF="mailto:michael@dayah.com">e-mail Michael Dayah</A>

e-mail Michael Dayah

Images

All linking is done with the A element followed by the HREF attribute. The only variation throughout the below sections is the value of the attribute HREF.

Retrieving

When you have found an image which you wish to save, hold down the mouse button and select "Save image as." Save the image to your desktop. Repeat as necessary.

Embedding

Use the IMG element's SRC attribute to embed an image on your page. Use the image's filename as the SRC attribute's value.

<IMG SRC="dogs.jpg">

Alignment

Images can be aligned left, right, or center. An especially nice effect can be obtained when IMG is used toward the beginning of a fairly long paragraph. Text will flow around the image depending on its alignment.
<IMG SRC="dogs.jpg" ALIGN="CENTER">

Alternate Text

Since not all people have a graphical browser, it is recommended that alternate text be given to an image using the ALT attribute. This text will also display while the image is loading.

<IMG SRC="dogs.jpg" ALT="Two dogs eating a cat">

Lists

Ordered

An ordered list is a list that is automatically numbered and aligned by the browser. The table of contents above is an ordered list. Use the OL element to contain an ordered list. Within the OL element are list items, LI.

<OL>
 <LI>Item one</LI>
 <LI>Item two</LI>
 <LI>Item three</LI>
</OL>

  1. Item one
  2. Item two
  3. Item three

Unordered

An unordered list is just like an ordered list, except it is bulleted instead of numbered. Its element is UL.

<UL>
 <LI>Item one</LI>
 <LI>Item two</LI>
 <LI>Item three</LI>
</UL>

  • Item one
  • Item two
  • Item three

Definition

A definition list, DL is an indented definition list. It has two sub-elements, DT (definition term) and DD (definition data). This page was laid out using nested definition lists.

<DL>
 <DT>Word one</DT>
  <DD>Definition one</DD>
 <DT>Word two</DT>
  <DD>Definition two</DD>
</DL>

Word one
Definition one
Word two
Definition two

Tables

Importance

It is close to impossible to really have a good, sharp, professional design without using tables. Tables are not just for showing data. Tables are the basic design element of all good web sites.

Basics

The TABLE element contains the entire table. Within the TABLE are table rows, TR. Within these rows are cells, TD (table data).

<TABLE>
 <TR>
  <TD>Row 1, Column 1</TD>
  <TD>Row 1, Column 2</TD>
 </TR>
 <TR>
  <TD>Row 2, Column 1</TD>
  <TD>Row 1, Column 2</TD>
 </TR>
</TABLE>

Row 1, Column 1Row 1, Column 2
Row 2, Column 1Row 1, Column 2

Borders

Notice that the in the above table, the borders are invisible. To make a table's borders visible, set the attribute BORDER to the size of the border in pixels.

<TABLE BORDER=5>
 <TR>
  <TD>Row 1, Column 1</TD>
  <TD>Row 1, Column 2</TD>
 </TR>
 <TR>
  <TD>Row 2, Column 1</TD>
  <TD>Row 1, Column 2</TD>
 </TR>
</TABLE>

Row 1, Column 1Row 1, Column 2
Row 2, Column 1Row 1, Column 2

Alignment

Table cells can be aligned vertically and horizontally. As with all other elements, the ALIGN attribute sets horizontal alignment. The ALIGN attribute of TD sets the horizontal alignment of text or images within the cell. The attribute VALIGN sets the vertical alignment of text or images within the cell. The value of VALIGN can be one of TOP, MIDDLE, or BOTTOM. Its default is MIDDLE.

<TABLE BORDER=1>
 <TR>
  <TD VALIGN="TOP">Left, Top</TD> (ALIGN="LEFT" is implied)
  <TD ALIGN="CENTER">Center, Middle</TD> (VALIGN="MIDDLE" is implied)
  <TD ALIGN="RIGHT" VALIGN="BOTTOM">Right, Bottom</TD>
 </TR>
</TABLE>

Left, TopCenter, MiddleRight, Bottom

Cell Coloring

A background color can be applied to cells or an entire table. Use BGCOLOR to set the background of a TD or TABLE. When the background color of the entire table is set, all cells in the table take that color as their background color unless the cell(s) already have a background color set.

<TABLE BGCOLOR="RED" BORDER=1> (BGCOLOR applies to entire table)
 <TR>
  <TD>One</TD>
  <TD>Two</TD>
  <TD BGCOLOR="GREEN">Three</TD> (TD's BGCOLOR overrides table's BGCOLOR)
 </TR>
</TABLE>
